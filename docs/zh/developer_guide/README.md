# 为 MindIE-SD 做贡献

感谢您有兴趣为 MindIE-SD 做出贡献！我们的社区对所有人开放，无论贡献大小，我们都欢迎各种形式的参与。您可以通过多种方式为该项目做出贡献，例如：

- 识别并报告任何问题或漏洞。
- 请求或添加对新模型的支持。
- 建议或实现新功能。
- 改进文档或撰写使用指南。

最后，支持我们最有影响力的方式之一是提高 MindIE-SD 的知名度。在您的博客中谈论它，并重点介绍它如何驱动您令人惊叹的项目。如果您正在使用 MindIE-SD，请在社交媒体上表达您的支持，或者只需为我们的仓库点赞（Star）以表示您的赞赏！

# 开发指南

## 开发环境构建

详情请参见[开发环境构建](./build_guide.md)

## 代码检查

MindIE-SD 使用 pre-commit 来对代码库进行 Lint 和格式化。详情请参见[Lint 与提交前检查](./tooling.md#lint-与提交前检查)

## 文档

MindIE-SD 使用 MkDocs 构建本地文档服务。详情请参见[本地文档服务构建](./tooling.md#本地文档服务构建)

## 测试

MindIE-SD 使用 unittest 来测试代码库。开发完成后，请补充完善相关的测试用例，并在提交PR前确保可以通过相关测试。运行测试用例，详情请参见[测试相关](./test.md)的本地文档服务构建章节
